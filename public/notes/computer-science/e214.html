Hello World

Java is a cross-platform, object-oriented language that runs in the Java Virtual Machine (JVM).

// Hello World Class
// In java the class name must match the file name
// this means that the file name must be HelloWorld.java
public class HelloWorld {
    // like C, Java also has a main method which is called 
    // when running the program as an executable
    // however unlike C, Java does not have a return type
    // the arguments also exclude the java command itself
    public static void main(String[] args) {
        // System is a class provided by the Java Development Kit (JDK)
        // it provides similar functionality to the C stdio library
        // here we're printing to the stdout (standard output)
        System.out.println("Hello World!");
    }
}

To compile this program, we must use a java compiler to generate a bytecode file. The standard java compiler is provided as a command line tool called javac. You must provide it with the name of the file to compile and it will output the bytecode file to the same directory as the source file.

Once compiled, it can be run in the JVM by using the java command.

% javac HelloWorld.java
% java HelloWorld

Builtin Types

Types tell the compiler how to interpret the values of variables, however they are still stored as bytes in the JVM memory.

  • Variable: a variable is a named location in memory that can hold a value of a certain type.
  • Literal: notation for expressing fixed value in source code.
  • Assignment: the process of assigning a value to a variable.
// this is a declaration statement for a variable called `x` of type `int`
int x;
// this is assignment statement, assigning a literal value of 1 to the variable `x`
x = 1;
// it can be combined into a single statement
int y = 1;

Java has 8 primative types: 1. int: 32-bit signed integer 2. long: 64-bit signed integer 3. float: 32-bit floating point 4. double: 64-bit floating point 5. boolean: 32-bit boolean 6. char: 16-bit Unicode character 7. byte: 8-bit signed integer 8. short: 16-bit signed integer

Java also provides a simple class for handling text: String.

Flow

Control Flow

  • Sequence of statements that are executed one after another.
  • Conditionals and loops are used to control the flow of the program. We can conditionally execute a statement or loop sequences based on a condition.

If Statement

Used for branching sequences of statements based on a condition. - If the condition is true, the first block of code is executed. - If the condition is false, the second block of code is executed.

// condition is a boolean expression
if (condition) {
    // code to execute if condition is true
} else {
    // code to execute if condition is false
    // you may omit this block
}

While Loop

Used for executing a sequence of statements while a condition is true.

// condition is a boolean expression
while (condition) {
    // code to execute while condition is true
}

Square Root Using a While Loop

Newton-Raphson method for finding the square root of a number \(c\).

  • initial guess: \(t_0 = c\)
  • \(t_n = \frac{t_{n-1} + \frac{2}{c}}{2}\)
  • repeat until desired precision \(\left|\frac{t_n - c}{t_n}\right| < t_n \cdot epsilon\)
public class SquareRoot {
    public static void main(String[] args) {
        // parseX is a helper function that parses a string to the desired type
        // we're reading from the commandline arguments
        double c = Double.parseDouble(args[0]);
        // the desired precision
        double epsilon = 0.01;
        // initial guess
        double t = c;
        // repeat until desired precision
        // relative error is difference between 
        //      current and next value divided by current value
        // we're using the absolute value of the relative error
        // and comparing it to the relative error of the current value
        while (Math.abs(t - c / t) > epsilon * t) {
            t = (t + c/t) / 2;
        }
        // print the result
        System.out.println(t);
    }
}

Do-While Loop

Less common than a while loop, but still useful. Checks the condition after executing the body.

do {
    // code to execute while condition is true

For Loop

Common looping construct.

  • Executes an initial statement
  • then evaluates an condition
  • if the condition is true, executes the body
  • then an increment statement
  • repeats until the condition is false.
for (int i = 0; /* init */  
     i < 10;    /* condition */
     i++)       /* increment */ 
{
    /* body */
}

Monte Carlo Simulation

According to the Wikipedia article:

Monte Carlo methods, or Monte Carlo experiments, are a broad class of computational algorithms that rely on repeated random sampling to obtain numerical results. The underlying concept is to use randomness to solve problems that might be deterministic in principle. They are often used in physical and mathematical problems and are most useful when it is difficult or impossible to use other approaches. Monte Carlo methods are mainly used in three problem classes:[1] optimization, numerical integration, and generating draws from a probability distribution.

basically, we use random numbers to simulate and compute the probability of a random event.

For example:

  • Gambler starts with stake and places $1 fair coin flip bets until going broke or reaching a goal.
  • How many times did they win?
public class Gambler {
    public static void main(String[] args) {
        // we're reading from the commandline arguments
        int stake = Integer.parseInt(args[0]);
        int goal = Integer.parseInt(args[1]);
        int trials = Integer.parseInt(args[2]);
        
        // repeat for each trial
        for (int t = 0; t < T; t++) {
            // do one gambler's ruin simulation
            int cash = stake;
            while (cash > 0 && cash < goal) {
                // generate a random number in [0, 1)
                double r = Math.random();
                if (r < 0.5) {
                    cash++;
                } else {
                    cash--;
                }
            }
            if (cash == goal) {
                // won the game
                wins++;
            }
        }
        // print the result
        System.out.println(wins + " wins of " + T);
    }
}

From statistics, the probability of winning is \(\frac{stake}{goal}\). This will more or less align with the monte carlo simulation.

We use monte carlo simulation to estimate the probability of a random event where doing so mathetically is impossible or inpractical.

Arrays

Indexed sequences of values of the same type.

int[] array = new int[10];  // creates a 10-element array of zeros

Java is a garbage-collected language, meaning that the garbage collector is responsible for reclaiming memory. You as programmer do not have to deallocate the array or the elements of the array (or any other data structure) manually.

  • Array indexing starts at 0.
  • Accessing an element of an array is done by using the index of the element with square brackets: array[index].

Shuffling an Array

Array elements can be swapped with each other using the below method.

// swap elements at indices i and j
void swap(int[] array, int i, int j) {
    int temp = array[i];
    array[i] = array[j];
    array[j] = temp;
}

Matrix Multiplication

// you may initialize the matrix with values as below
double[][] a = {{1, 2}, {3, 4}};
double[][] b = {{5, 6}, {7, 8}};
double[][] c = new double[2][2];
for (int i = 0; i < 2; i++) {
    for (int j = 0; j < 2; j++) {
        for (int k = 0; k < 2; k++) {
            c[i][j] += a[i][k] * b[k][j];
        }
    }
}
// this is O(n^3) time and O(n^2) space

Redirection and Piping

Your terminal shell provides a way to redirect the input/output of a command to a file. It also allows for piping of data between two commands.

This will redirect standard output to the file output.txt.

In this command, the output of the Add command is piped to the input of the Square command.

Standard Audio

Sound is the perception of the vibration of particles in a medium. This motion curve can be represented by sampling the sound wave at a regular interval.

We must sample at twice the maximum frequency we wish to capture in able to capture the entire wave form without losing much information.

For the human ear this maximum frequency is 22.05kHz, so we sample at 44.1kHz.

\(y(i) = \sin{\frac{2\pi \cdot i \cdot 440}{44100}}\)

Our course allows us to play these samples using the StdAudio class.

public class StdAudio {
    // play wav file
    void play(String file); 
    // play a 44.1kHz waveform encoded as 64-bit floating point audio samples
    void play(double[] a);
    // play a single sample
    void play(double a);
    // save a wav file
    void save(String file, double[] a);
    // read a wav file to a double array
    double[] read(String file);
}

When adding mutiple sounds together, we can simply add the values of the samples together but then have to normalize the values back down by dividing by the number of samples we added together.

Functions

Funtions take several arguments as input and return a value.

Unlike mathematical functions, programming functions may get information from additional sources, not just the provided arguments. When the function alters state outside of its scope, the function is said to have side effects. It is common for functions which use side effects to be called mutators and return void - a special type for returning no value.

Functions are useful for building modular programs.

Weve seen some built-in functions like Math.random(), but we can also define our own functions like main.

// define a function
// this first line is called the signature
// it informs us of the return type, the name of the function
// and the types of the parameters
public static double sqrt(double c) {
    // arguments are treated as local variables to this scope
    // local variables can only be accessed within the function scope
    // and its children scopes
    double err = 1e-15;
    double t = c;
    while (Math.abs(t - c/t) > err * t) {
        // the variables err, t and c are still accessible here
        // since they are declared in the function scope
        // which is the parent of this block scope
        t = (c/t + t) / 2.0;
    }
    // we return a value using the return keyword
    return t;
}

Functions allow for a new way to control the flow of a program. We can jump to different sequences of code in our program by calling functions.

When a function is called, the program jumps to the first line of the function and then executes the function. Afterwards it will return the the program to the line after the function call. This is possible by using a stack data structure, in which the program stores the current state of the program and then jumps to the next line of code, afterwards we can pop the top of the stack and return to the previous state.

Java does not support explicitly stating whether to use call by value or call by reference. This is because the compiler will automatically determine the correct behavior. For primative types, the compiler will use call by value. For objects, the compiler will use call by reference. Therefore, when mutating a object in the function scope, it will also mutate the object in the caller scope.

Java supports method overloading. This means that we can define multiple functions with the same name but different parameters and the compiler will automatically select the correct function to call based off the arguments provided.

Functions allow for abstraction. We can make use of functions to hide the implementation of a specific feature, which can then in turn be used as building blocks for more functions and abstractions.

Libraries

  • Library: A module of code that is designed to be used by other programs.
  • Client: A program that uses a library.
  • API: Tells the client about the functions available in the library without revealing the implementation. Typically it will provided the function signatures.
  • Implementation: The code that actually implements the methods of the API.

The Java compiler will use an environment called the classpath to find libraries. The classpath is a list of directories that the compiler will search for libraries.

Java provides the concept of packages to for grouping classes together like a folder in a file system and you only need the base folder in the classpath to access the entire package.

Javas API is called the JDK or Java Development Kit and provides the base libraries for the Java language on which all other libraries are based.

Test Driven Development

Test Driven Development is a software development process that uses a test suite to validate the correctness of a program.

In test driven development, the programmer writes tests first, then the program is developed to pass the tests.

We may use the main method to provide a test suite for our program or we may use some framework like JUnit to provide a test suite for our program.

Modular Programming

  • Divide program into self-contained pieces.
  • Test each piece independently.
  • Combine pieces to create a larger program.
  • Simplify building of complex programs by breaking them into smaller, reusable pieces.
  • Improves debugging and maintenance.

Recursion

When a function calls itself, directly or indirectly, it is called recursion. Recursion is a powerful programming paradigm.

Recursion is closely related to mathematical induction, proving something is true for a large case and assuming the same for a smaller case. It has a base case and inductive step.

Recursion works in the opposite direction, breaking the problem into smaller parts instead of bigger parts, also having a base case but instead of inductive step has a recursive step to make the problem smaller.

Greatest Common Divisor

Lets take for example the greatest common divisor of two numbers.

We can compute this using euclidean algorithm.

\(gcd(a, b) = \Bigg \{ \begin{array}{ll} b & \text{if } a = 0 \\ gcd(b, a \mod b) & \text{otherwise} \end{array}\)

Here \(b \text{ if } a = 0\) is the base case and \(gcd(b, a \mod b)\) is the recursive step.

We can implement it in Java as follows:

public static int gcd(int a, int b) {
    if (b == 0) return a;
    return gcd(b, a % b);
}

Fibonacci Sequence

\(F(n) = \Bigg \{ \begin{array}{ll} 0 & \text{if } n = 0 \\ 1 & \text{if } n = 1 \\ F(n - 1) + F(n - 2) & \text{otherwise} \end{array}\)

The recursive java function looks like this:

public static long fib(int n) {
    if (n == 0) return 0;
    if (n == 1) return 1;
    return fib(n - 1) + fib(n - 2);
}

This function is very slow and inefficient, recomputing the same values over and over again.

We may use a memoization without recursion to improve the performance of the function.

public static long fib(int n) {
    if (n == 0) return 0;
    long[] F = new long[n + 1];
    F[0] = 0;
    F[1] = 1;
    for (int i = 2; i <= n; i++) {
        F[i] = F[i - 1] + F[i - 2];
    }
    return F[n];
}
